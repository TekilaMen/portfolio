Mise en situation

Universalis est un projet mené durant mon temps libre à mon domicile, voici le contexte :
le jeu Final Fantasy XIV est composé de deux datacenters contenants eux-mêmes plusieurs mondes. Un hôtel des ventes a été mis en place récemment permettant à ses joueurs de mettre en ventes différents types d’objets pour les personnes d’un même monde, cependant il est actuellement impossible de vérifier les prix des autres mondes et datacenters sans se déplacer directement dans celui-ci. Pour régler cela FF14 met en place une web api nommé Universalis pour faciliter cela.

Documentation

Cahier des charges

Le cahier des charges nous demande de développer une application web simple permettant :
- D'afficher la liste de tous les objets avec leur id,
- D'afficher les cinq derniers objets mis à jours récement par datacenters,
- De pouvoir rechercher un objet par son id, datacenter et monde, puis d'afficher les neuf vendeurs ayant le prix le moins cher.

En respectant certaines attentes qui sont composées de :
- Une interface intuitive et simple d'utilisation,
- Le respect de la règle des 3 clics (Fonction de recherche non comprise).

Avis & Finalité

Ce projet m'a permis de découvrir en profondeur l'utilisation d'un web api et d'utiliser les données fournies avec du PHP. En effet, ne possédant pas un niveau énorme en PHP les débuts furent assez compliqués, mais avec de la persévérance et de la documentation j'ai pu mener à bien ce projet tout en trouvant cette expérience très amusante.
